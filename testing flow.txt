Bagus, ini langkah yang benar 👍 Sekarang kita masuk ke **QA (Quality Assurance) / testing flow** supaya kamu bisa yakin semua fitur sudah sesuai soal.

Aku bikin seperti checklist + cara ngetesnya biar kamu bisa “simulasi perusahaan”.

---

# 🧪 🧾 QA FLOW PROJECT KAMU

Kita test dari awal sampai akhir:

---

# 1. 🟢 TEST SERVER & DATABASE

## ✅ Cek server

```bash id="qa1"
npm run dev
```

Harus muncul:

```text id="qa2"
Server running on port 3000
Database connected
```

---

# 2. 🔐 TEST LOGIN

## Endpoint:

```http id="qa3"
POST /auth/login
```

## Body:

```json id="qa4"
{
  "username": "admin",
  "password": "123456"
}
```

---

## ✔ Expected hasil:

### Kasus A (1 role):

```json id="qa5"
{
  "token": "JWT_TOKEN"
}
```

---

### Kasus B (multi role):

```json id="qa6"
{
  "message": "Pilih role",
  "roles": [...]
}
```

---

# 3. 🔀 TEST PILIH ROLE

## Endpoint:

```http id="qa7"
POST /auth/select-role
```

## Body:

```json id="qa8"
{
  "user_id": 1,
  "role_id": 1
}
```

---

## ✔ Expected:

```json id="qa9"
{
  "token": "JWT_TOKEN_BARU"
}
```

---

# 4. 🔑 TEST AUTH TOKEN (WAJIB)

## Endpoint:

```http id="qa10"
GET /menus
```

## Header:

```text id="qa11"
Authorization: Bearer <TOKEN>
```

---

## ❌ Jika salah:

```json id="qa12"
{"message":"Token tidak ditemukan"}
```

## ❌ Jika invalid:

```json id="qa13"
{"message":"Token tidak valid"}
```

## ✔ Jika benar:

```json id="qa14"
[
  {
    "id": 1,
    "name": "Dashboard"
  }
]
```

---

# 5. 🌳 TEST MENU ACCESS (ROLE BASED)

Coba login dengan role berbeda:

## Admin:

✔ lihat semua menu

## User:

✔ hanya menu terbatas

👉 Ini bukti sistem RBAC kamu berhasil

---

# 6. 🧩 TEST DATABASE INTEGRITY

Di PostgreSQL:

```sql id="qa15"
SELECT * FROM users;
SELECT * FROM roles;
SELECT * FROM user_roles;
SELECT * FROM menus;
SELECT * FROM role_menus;
```

---

# 7. 🔥 TEST EDGE CASE (INI PENTING UNTUK NILAI)

## ❌ Salah password

→ harus reject

## ❌ User tidak ada

→ harus 404

## ❌ Token expired

→ harus 401

## ❌ Role tidak dipilih

→ tidak boleh akses menu
